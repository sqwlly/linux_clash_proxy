# 状态输出着色设计

**目标**

在不破坏脚本消费和终端兼容性的前提下，为 `status` 套件增加功能性色彩，提升信息层级与扫读效率。

**设计原则**

- KISS：只给区块标题、状态词、少量关键值上色。
- 兼容优先：默认仅在 TTY 输出启用；`FORCE_COLOR=1` 可强制开启。
- `--raw` 不加颜色。
- Python CLI 与 `proxy.sh` 保持同一配色语义。

**配色方案**

- 蓝色加粗：区块标题，如 `摘要`、`资源`、`结果`
- 绿色：`正常`、`运行中`、`可访问`
- 黄色：`部分异常`、`待刷新`、`未知`
- 红色：`失败`、`异常`、`未运行`、`不可访问`
- 青色：少量关键值，如 `当前选择`、`当前出口`、`出口 IP`

**实现范围**

- `status`、`ai-status`、`list-groups`、`list-nodes`、`test-group`、`test`
- `current`、`switch`、`logs`
- 增加 `FORCE_COLOR=1` 回归测试

**非目标**

- 不做复杂渐变、边框或 TUI
- 不给整行大面积着色
- 不修改原有命令语义
